# Система управления переговорными комнатами

Веб-приложение для бронирования и управления переговорными комнатами в офисах и коворкингах.

Проект разработан командой студентов в рамках учебной практики. В отличие от обычного календаря бронирования, система рассматривается как основа платформы управления офисным пространством, объединяющей **цифровой след** (информация о бронировании) с **физическим состоянием** переговорных комнат (фактическим присутствием людей).

---

## ✨ Возможности

* Регистрация и авторизация пользователей
* Недельный календарь бронирования переговорных комнат
* Создание и проверка корректности бронирований
* Страница текущей занятости переговорных
* Панель администратора (Dashboard)
* Статистика и аналитика использования переговорных
* Поддержка светлой и тёмной темы оформления
* Адаптивный веб-интерфейс
* Клиент-серверная архитектура на основе REST API

## 🏗 Архитектура

```
                  Веб-браузер
                       │
                       ▼
         HTML + CSS + JavaScript
                       │
       REST API (JSON поверх HTTP)
                       │
                       ▼
              Сервер Flask (Python)
                       │
                 SQLAlchemy ORM
                       │
                       ▼
                 База данных MySQL
```

Архитектура построена по классической клиент-серверной модели. Пользователь взаимодействует с веб-интерфейсом, который отправляет HTTP-запросы к REST API. Сервер Flask обрабатывает запросы, выполняет бизнес-логику приложения и обращается к базе данных MySQL посредством ORM SQLAlchemy.

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/codyfine/meeting-room-management.git
cd meeting-room-management
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Создание базы данных

Запустите SQL-скрипт

```
database/db_create.sql
```

Скрипт автоматически:

* удалит существующую базу данных (если она уже создана);
* создаст новую базу данных;
* создаст все необходимые таблицы;
* подготовит структуру базы данных для работы приложения.

### 4. Настройка подключения к базе данных

При необходимости измените параметры подключения к MySQL в файле `app.py`.

### 5. Запуск приложения

```bash
python app.py
```

После запуска приложение будет доступно по адресу

```
http://localhost:5000
```
